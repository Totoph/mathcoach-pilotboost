# 2026-03-02 - Tests locaux & fixes critiques

## Session de debug (00:00 - 00:25)

### Problème initial
- **Erreur 422** sur `/api/v1/agent/submit-answer`
- Frontend envoyait `undefined` pour `correct_answer`
- Backend exigeait ce champ dans le schema Pydantic

### Cause racine
Le backend ne renvoyait jamais `correct_answer` au frontend (logique : on ne veut pas tricher), mais exigeait que le frontend le renvoie lors du submit → champ manquant → **validation error 422**.

### Solution implémentée
1. **Cache serveur des exercices** (`_exercise_cache` dict)
   - Stocke `correct_answer` + métadonnées côté backend lors de `next-exercise`
   - Récupère automatiquement ces données lors du `submit-answer`
   - Supprime l'entrée après usage (one-time use)

2. **Schema simplifié**
   - `SubmitAnswerRequest` : seulement `exercise_id`, `user_answer`, `time_taken_ms`
   - Plus besoin d'envoyer `question`, `correct_answer`, `exercise_type`, etc. depuis le frontend

3. **Sécurité améliorée**
   - Le client ne peut plus manipuler `correct_answer`
   - Toute la validation se fait côté serveur

### Problèmes UX identifiés
1. **Pas de barre de progression** → utilisateur perdu
2. **Diagnostic jamais complété** → logique manquante

### Fixes UX implémentés

#### Backend
- **Complétion automatique du diagnostic** après 10 exercices
- **Calcul du niveau** (1-5) basé sur :
  - Difficulté des derniers exercices
  - Nombre de faiblesses identifiées
- **Message de félicitation** automatique à la fin du diagnostic

#### Frontend
- **Barre de progression** : `Diagnostic : X/10` avec barre visuelle
- **Affichage post-diagnostic** : `Niveau X/5 · Y exercices`
- **État synchronisé** après chaque soumission
